# App Data — Curated Workout Dataset

All valuable data extracted from the 258 curated workouts (output of the strict
multi-combo review), organized for direct use in the fitness app.
Generated by `extract_app_data.py`.

## Three app categories

Every workout is placed in exactly one category:

| Category | Count | Meaning | App usage |
|----------|-------|---------|-----------|
| `full_workout` | 112 | Ongoing, repeatable routines (PHUL, dumbbell split, 3-day full body…). Most state a suggested run length in metadata, but they're designed to be followed indefinitely. | "Workouts" — pick one and run it week to week |
| `program` | 71 | Complete plans with an explicit fixed duration in the title/description ("8 Week Mass Building…", "12 Week Women's Trainer"). Clear start → finish journey. | "Programs" — enroll, track progress to completion |
| `focused` | 75 | Single-muscle-group / focus workouts (chest day, glute builder, ab circuit). Add-ons to an existing routine, tagged with `muscle_focus`. | "Focused workouts" — browse by muscle |

Categorization rules:
- `workout_type == single muscle group` → **focused** (1 manual correction:
  `sculpted-strength-bodybuilding-program` is actually a 6-day full plan).
- else, explicit fixed length stated in title/description → **program**.
- else → **full_workout**.

### `schedule` structure

```json
"schedule": [
  {
    "label": "Day 1 - Upper Power",
    "exercises": [
      {"exercise": "Barbell Bench Press",
       "exercise_url": "https://www.muscleandstrength.com/exercises/barbell-bench-press.html",
       "sets": "3-4", "reps": "3-5"}
    ]
  }
]
```

- Optional per-exercise fields when the source provides them: `rest`, `tempo`,
  `time`, `sets_x_reps`, `group` (superset/circuit grouping).
- Some sections use `notes` instead of `exercises` (instruction-style tables,
  e.g. "Day 1: perform each movement 3×15…").
- 1 workout has no extractable schedule
  (`dumbbell-bodyweight-circuit-workout-women` — content not in the HTML).

## Exercise library

`exercise_library.csv`: 6,080 exercise slots collapsed into **1,164 canonical
exercises** (canonicalized by M&S exercise-page slug; 457 carry a link to the
exercise page with instructions). `aliases` lists every raw name variant seen,
useful for search/matching.

## Exercise calories (exercise_calories.csv)

Calorie data joined from FitClick's calculator
([fitclick.com/calories_burned](https://www.fitclick.com/calories_burned)),
scraped at the site defaults: **Body Weight 150 lbs, Exercise Duration 15 min**
(all 6,099 FitClick exercises captured in `fitclick_exercises_raw.json`).

Per canonical exercise:
- `cal_per_hr_150lb` — FitClick burn rate at 150 lbs for 60 min
- `cal_15min_150lb` — the default 15-min value (`cal_per_hr / 4`)
- `met_estimate` — `cal_per_hr_150lb / 71.4`
- `calorie_source` — `fitclick_match` (522 exercises; 86% of all workout
  exercise slots) \| `category_estimate` (body-part median) \|
  `strength_median_estimate` (overall strength median, 408 cal/hr)
- match metadata: `fitclick_name`, `fitclick_eqID`, `fitclick_url`,
  `fitclick_category`, `fitclick_muscle`, `match_type`, `match_score`

**Calculate for any body weight and duration** (FitClick's output is verified
linear in both):

```
calories = cal_per_hr_150lb × (body_weight_lb / 150) × (duration_min / 60)
calories = cal_per_hr_150lb × (body_weight_kg / 68.04) × (duration_min / 60)
```

Equivalently via METs: `calories/min = MET × 3.5 × kg / 200` with
`MET = cal_per_hr_150lb / 71.4`.
Example: Squat (442 cal/hr) for a 180 lb user, 20 min →
`442 × 180/150 × 20/60 ≈ 177 cal`. Full details in `calories_formula.json`.
Rebuild with `build_exercise_calories.py`.
